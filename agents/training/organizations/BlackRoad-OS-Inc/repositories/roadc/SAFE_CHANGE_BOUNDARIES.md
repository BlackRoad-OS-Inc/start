# roadc — Safe Change Boundaries

## SAFE (do freely)
- Read all source files
- Write and run RoadC programs
- Run tests

## CAUTION (test first, verify after)
- Modify the parser grammar
- Change the interpreter
- Edit the lexer

## DANGEROUS (ask before doing)
- Change the language syntax (breaks existing programs)
- Modify the C compiler backend
- Change the REPL interface

## NEVER
- Break backward compatibility silently
- Remove language features without deprecation
- Force push to main
