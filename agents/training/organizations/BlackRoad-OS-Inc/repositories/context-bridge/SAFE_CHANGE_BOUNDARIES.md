# context-bridge — Safe Change Boundaries

## SAFE (do freely)
- Read source code and tests
- Run tests locally
- Use CLI for testing

## CAUTION (test first, verify after)
- Modify memory storage schema
- Change the Chrome extension
- Edit CLI commands

## DANGEROUS (ask before doing)
- Deploy the Chrome extension (public)
- Change Worker endpoints
- Modify context encryption

## NEVER
- Store user context unencrypted
- Expose raw context in API responses
- Force push to main
