# remember — Safe Change Boundaries

## SAFE (do freely)
- Read source code and tests
- Store/recall test memories
- Edit documentation

## CAUTION (test first, verify after)
- Modify the symbolic kernel
- Change vector storage schema
- Edit contradiction logic

## DANGEROUS (ask before doing)
- Change the memory format (breaks existing stores)
- Modify Lucidia integration
- Delete memory databases

## NEVER
- Delete user memory data without consent
- Store unencrypted sensitive data
- Force push to main
