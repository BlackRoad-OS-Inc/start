# ai-chain — Safe Change Boundaries

## SAFE (do freely)
- Read source code and tests
- Run tests locally
- Edit documentation

## CAUTION (test first, verify after)
- Modify routing/failover logic
- Change node configuration
- Add new API endpoints

## DANGEROUS (ask before doing)
- Deploy to Pi fleet
- Change node addresses or ports
- Modify Ollama API integration

## NEVER
- Hardcode node IP addresses
- Remove failover logic
- Force push to main
