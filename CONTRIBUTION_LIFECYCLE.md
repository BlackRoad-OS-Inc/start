# Contribution Lifecycle

## The Path

```
Discover → Certify → Claim → Build → Test → Review → Deploy → Log → Teach
```

## Step by Step

### 1. Discover
- Search [CODEX] for existing solutions
- Check [TIL] feed for recent learnings
- Read the repo's curriculum in Start
- Identify the work to be done

### 2. Certify
- Complete Observer certification for the target repo
- Verify you understand the architecture
- Know the safe change boundaries

### 3. Claim
- Assign the issue to yourself
- Log your intent to [JOURNAL]
- Check [TODO] for related open work

### 4. Build
- Follow coding standards (governance/policies/coding.json)
- Stay within safe change boundaries
- Stop when uncertain — ask before destructive actions

### 5. Test
- Run existing tests
- Add tests for new functionality
- Test locally before any remote action

### 6. Review
- Submit PR with Summary + Test Plan
- All PRs require @blackboxprogramming approval
- Address review feedback

### 7. Deploy
- Deploy via `wrangler deploy` or appropriate tool
- Verify with health check
- Monitor for errors

### 8. Log
- Log completion to [JOURNAL]
- Mark [TODO] complete
- Update [CODEX] if you solved something new

### 9. Teach
- Broadcast [TIL] with what you learned
- Update the repo curriculum if applicable
- Help the next agent who works on this
